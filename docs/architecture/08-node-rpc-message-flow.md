# Node 간 RPC 메시지 처리 플로우

## 개요

Service Mesh (Control Plane)를 통한 Node 간 RPC 통신 전체 과정을 상세히 설명합니다.

**예시 시나리오**: GameServer A가 Gateway에게 세션 Pin 요청

---

## 📤 Phase 1: 요청 발신 (GameServer A → Gateway)

### 1.1 Controller에서 RPC 호출
```csharp
// GameServer A의 SessionController.cs
public async Task<SessionResponse> LoginAsync(LoginRequest req)
{
    // ... 로그인 처리 ...

    // Gateway에 세션 Pin 요청 (Node RPC)
    var pinReq = new ServiceMeshPinSessionReq
    {
        SocketId = socketIdBytes,
        SessionId = sessionId,
        SourceGameServerId = _myNodeId
    };

    // INodeSender를 통한 RPC 호출
    var pinRes = await _nodeSender.RequestAsync<
        ServiceMeshPinSessionReq,
        ServiceMeshPinSessionRes
    >(
        actorId: NodePacket.ServerActorId,  // 0 = Stateless
        targetNodeId: gatewayNodeId,
        message: pinReq
    );

    return Response.Ok(new SessionResponse { ... });
}
```

### 1.2 INodeSender.RequestAsync 내부
```csharp
// SimpleNetEngine.Node/Network/INodeSender.cs
public async Task<TResponse> RequestAsync<TRequest, TResponse>(
    long actorId, long targetNodeId, TRequest message)
{
    // 1. RequestKey 생성 (응답 매칭용)
    int requestKey = _requestKeyGenerator.Next();

    // 2. NodePacket 생성
    var packet = NodePacket.Create(
        actorId: actorId,
        srcId: _myNodeId,
        destId: targetNodeId,
        requestKey: requestKey,
        isReply: false,
        message: message
    );

    // 3. RequestCache에 TaskCompletionSource 등록
    var tcs = new TaskCompletionSource<NodePacket>();
    _requestCache.Add(requestKey, tcs);

    // 4. NetMQ Router로 전송
    _nodeCommunicator.Send(targetNodeId, packet);

    // 5. 응답 대기 (Task await)
    var responsePacket = await tcs.Task;

    // 6. 응답 파싱
    var parser = AutoGeneratedParsers.GetParserById(responsePacket.Header.MsgId);
    return (TResponse)parser.ParseFrom(responsePacket.Payload);
}
```

### 1.3 NetMQ 전송
```csharp
// SimpleNetEngine.Node/Network/NodeCommunicator.cs
public void Send(long targetNodeId, NodePacket packet)
{
    // 1. NetMQ Msg 준비
    var msg = new Msg();
    msg.InitPool(packet.TotalSize);

    // 2. Identity Frame (Router 라우팅용)
    var identityBytes = BitConverter.GetBytes(targetNodeId);
    _socket.SendMoreFrame(identityBytes);

    // 3. NodePacket 전송
    _socket.SendFrame(msg);
}
```

**🔹 Phase 1 요약**:
```
Controller → INodeSender.RequestAsync()
           → NodePacket 생성
           → RequestCache에 대기 TaskCompletionSource 등록
           → NetMQ Router.Send(targetNodeId, packet)
           → [await 응답]
```

## 📥 Phase 2: 요청 수신 및 처리 (Gateway)

### 2.1 NetMQ 수신
```csharp
// SimpleNetEngine.Node/Network/NodeCommunicator.cs (Gateway 측)
private void OnReceiveReady(object? sender, NetMQSocketEventArgs e)
{
    // 1. Identity Frame 읽기 (발신자 NodeId)
    var identityMsg = new Msg();
    _socket.ReceiveMsg(ref identityMsg);
    long senderNodeId = BitConverter.ToInt64(identityMsg);

    // 2. NodePacket Frame 읽기
    var msg = new Msg();
    _socket.ReceiveMsg(ref msg);

    // 3. NodePacket 파싱
    var packet = NodePacket.Create(ref msg);

    // 4. EventController로 라우팅
    _eventController.OnPacket(packet);
}
```

### 2.2 GatewayNodeEventController.OnPacket()
```csharp
// SimpleNetEngine.Gateway/Core/GatewayNodeEventController.cs
public override void OnPacket(NodePacket internalPacket)
{
    using (internalPacket)
    {
        // 1. ActorMessageFactory로 파싱
        var actorMessage = _actorMessageFactory.Create(internalPacket);

        // ActorMessage {
        //   Header: NodeHeader (ActorId, Source, Dest, MsgId, RequestKey)
        //   Message: ServiceMeshPinSessionReq (파싱된 Protobuf)
        //   RawPayload: ReadOnlyMemory<byte> (원본 bytes)
        // }

        if (actorMessage != null)
        {
            // 2. base.Push() → QueuedResponseWriter<ActorMessage>
            Push(actorMessage);
        }
    }
}
```

### 2.3 QueuedResponseWriter 순차 처리
```csharp
// SimpleNetEngine.Node/Core/QueuedResponseWriter.cs
private readonly Channel<T> _channel = Channel.CreateUnbounded<T>();

public void Write(T message)
{
    _channel.Writer.TryWrite(message);
}

private async Task ProcessQueueAsync()
{
    await foreach (var message in _channel.Reader.ReadAllAsync())
    {
        try
        {
            // DispatchAsync 호출 (sequential 보장)
            await _dispatchFunc(message);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Error in QueuedResponseWriter");
        }
    }
}
```

### 2.4 GatewayNodeEventController.DispatchAsync()
```csharp
// SimpleNetEngine.Gateway/Core/GatewayNodeEventController.cs
protected override async Task DispatchAsync(ActorMessage message)
{
    IMessage? response = null;
    try
    {
        // 1. Scoped DI 생성
        await using var scope = _scopeFactory.CreateAsyncScope();

        // 2. ActorMessage → NodePacket 재구성
        using var reconstructedPacket = NodePacket.Create(
            message.Header.ActorId,
            message.Header.Source,
            message.Header.Dest,
            message.Header.RequestKey,
            message.Header.IsReply != 0,
            message.Message
        );

        // 3. INodeDispatcher로 디스패치
        response = await _dispatcher.DispatchAsync(
            scope.ServiceProvider,
            reconstructedPacket
        );
    }
    catch (Exception e)
    {
        _logger.LogError(e, "Error processing node packet");
    }

    // 4. 응답 전송
    if (response != null)
        _responser.Response(message.Header, response);
}
```

### 2.5 INodeDispatcher.DispatchAsync()
```csharp
// SimpleNetEngine.Node/Core/NodeDispatcher.cs
public async Task<IMessage?> DispatchAsync(IServiceProvider scope, NodePacket packet)
{
    // 1. MsgId로 핸들러 조회
    if (_handlers.TryGetValue(packet.Header.MsgId, out var handler))
    {
        // handler: Func<IServiceProvider, NodePacket, Task<IMessage?>>

        // 2. 핸들러 실행
        return await handler(scope, packet);
    }

    _logger.LogWarning("No handler for MsgId={MsgId}", packet.Header.MsgId);
    return null;
}
```

**핸들러는 어떻게 등록되었나?**
```csharp
// GatewaySample/Program.cs
services.AddNodeControllers();  // 자동 스캔 + 등록
```

```csharp
// SimpleNetEngine.Node/Extensions/NodeControllerExtensions.cs
public static IServiceCollection AddNodeControllers(this IServiceCollection services)
{
    // 1. Assembly에서 [NodeController] 스캔
    var controllerTypes = assembly.GetTypes()
        .Where(t => t.GetCustomAttribute<NodeControllerAttribute>() != null);

    // 2. Controller를 Scoped로 등록
    foreach (var type in controllerTypes)
        services.AddScoped(type);

    // 3. INodeDispatcher 등록 + 핸들러 자동 등록
    services.AddSingleton<INodeDispatcher>(sp =>
    {
        var dispatcher = new NodeDispatcher(logger);

        foreach (var controllerType in controllerTypes)
        {
            // [NodePacketHandler] 메서드 찾기
            var methods = controllerType.GetMethods()
                .Where(m => m.GetCustomAttribute<NodePacketHandlerAttribute>() != null);

            foreach (var method in methods)
            {
                var attr = method.GetCustomAttribute<NodePacketHandlerAttribute>();
                var msgId = attr.MsgId;

                // 핸들러 래퍼 생성
                dispatcher.RegisterHandler(msgId, async (provider, packet) =>
                {
                    // 파싱
                    var parser = AutoGeneratedParsers.GetParserById(msgId);
                    var message = parser.ParseFrom(packet.Payload);

                    // Controller 인스턴스 생성 (Scoped)
                    var controller = provider.GetRequiredService(controllerType);

                    // 메서드 호출
                    var task = (Task<IMessage>)method.Invoke(controller, [message]);
                    return await task;
                });
            }
        }

        return dispatcher;
    });
}
```

### 2.6 Controller 핸들러 실행
```csharp
// GatewaySample/Controllers/ControlController.cs
[NodeController]
public class ControlController(
    ILogger<ControlController> logger,
    ConcurrentDictionary<Guid, GatewaySession> clientSessions,
    SessionMapper sessionMapper)
{
    [NodePacketHandler(ServiceMeshPinSessionReq.MsgId)]
    public async Task<IMessage> PinSession(ServiceMeshPinSessionReq req)
    {
        var socketId = new Guid(req.SocketId.Span);

        if (clientSessions.TryGetValue(socketId, out var session))
        {
            // 세션 Pin 처리
            session.Pin(req.SourceGameServerId, req.SessionId);
            sessionMapper.PinSocket(socketId, req.SourceGameServerId, req.SessionId);

            logger.LogInformation(
                "Session Pinned: SocketId={SocketId}, SessionId={SessionId}",
                socketId, req.SessionId);

            // 성공 응답
            return new ServiceMeshPinSessionRes { Success = true };
        }

        // 실패 응답
        return new ServiceMeshPinSessionRes { Success = false };
    }
}
```

**🔹 Phase 2 요약**:
```
NetMQ Router.OnReceiveReady()
  → NodeCommunicator.OnReceiveReady()
  → NodePacket 파싱
  → GatewayNodeEventController.OnPacket()
  → ActorMessageFactory.Create() (NodePacket → ActorMessage)
  → base.Push(actorMessage) → QueuedResponseWriter<ActorMessage>
  ↓ (Sequential Queue)
  → GatewayNodeEventController.DispatchAsync()
  → ActorMessage → NodePacket 재구성
  → INodeDispatcher.DispatchAsync()
  → 등록된 Handler 조회
  → ControlController.PinSession() 실행
  → ServiceMeshPinSessionRes 반환
```

## 📤 Phase 3: 응답 전송 (Gateway → GameServer A)

### 3.1 INodeResponser.Response()
```csharp
// SimpleNetEngine.Node/Network/INodeResponser.cs
public void Response(NodeHeader requestHeader, IMessage responseMessage)
{
    // 1. 응답 NodePacket 생성
    var responsePacket = NodePacket.Create(
        actorId: requestHeader.ActorId,
        srcId: requestHeader.Dest,      // 발신자 = 현재 노드
        destId: requestHeader.Source,   // 수신자 = 요청한 노드
        requestKey: requestHeader.RequestKey,  // 매칭용 키
        isReply: true,
        message: responseMessage
    );

    // 2. NetMQ 전송
    _nodeCommunicator.Send(requestHeader.Source, responsePacket);
}
```

**🔹 Phase 3 요약**:
```
ControlController.PinSession() 반환
  → GatewayNodeEventController.DispatchAsync()에서 response 수신
  → INodeResponser.Response(header, response)
  → 응답 NodePacket 생성 (isReply=true, requestKey 포함)
  → NetMQ Router.Send(sourceNodeId, responsePacket)
```

## 📥 Phase 4: 응답 수신 및 완료 (GameServer A)

### 4.1 NetMQ 응답 수신
```csharp
// SimpleNetEngine.Node/Network/NodeCommunicator.cs (GameServer A 측)
private void OnReceiveReady(object? sender, NetMQSocketEventArgs e)
{
    // NodePacket 파싱
    var packet = NodePacket.Create(ref msg);

    // IsReply 체크
    if (packet.Header.IsReply != 0)
    {
        // 응답 패킷 → RequestCache로 전달
        _requestCache.Complete(packet.Header.RequestKey, packet);
    }
    else
    {
        // 요청 패킷 → EventController로 라우팅
        _eventController.OnPacket(packet);
    }
}
```

### 4.2 RequestCache 완료
```csharp
// SimpleNetEngine.Infrastructure/RequestCache.cs
public void Complete(int requestKey, NodePacket responsePacket)
{
    if (_pendingRequests.TryRemove(requestKey, out var tcs))
    {
        // TaskCompletionSource 완료
        tcs.SetResult(responsePacket);
    }
}
```

### 4.3 INodeSender.RequestAsync 재개
```csharp
// Phase 1.2에서 await 중이던 곳
var responsePacket = await tcs.Task;  // ← 여기서 재개!

// 응답 파싱
var parser = AutoGeneratedParsers.GetParserById(responsePacket.Header.MsgId);
var response = (ServiceMeshPinSessionRes)parser.ParseFrom(responsePacket.Payload);

return response;  // Controller로 반환
```

**🔹 Phase 4 요약**:
```
NetMQ Router.OnReceiveReady() (GameServer A)
  → NodePacket 파싱 (isReply=true 확인)
  → RequestCache.Complete(requestKey, responsePacket)
  → TaskCompletionSource.SetResult()
  → await 중이던 INodeSender.RequestAsync() 재개
  → 응답 파싱 및 반환
  → Controller로 결과 전달
```

## 📊 핵심 컴포넌트 역할

### 발신 측 (Sender)
| 컴포넌트 | 역할 |
|---------|------|
| **Controller** | 비즈니스 로직, RPC 호출 |
| **INodeSender** | RPC 요청 생성, RequestCache 관리, await 응답 |
| **NodeCommunicator** | NetMQ Router 송수신 |
| **RequestCache** | RequestKey → TaskCompletionSource 매핑 |

### 수신 측 (Receiver)
| 컴포넌트 | 역할 |
|---------|------|
| **NodeCommunicator** | NetMQ 수신, NodePacket 파싱 |
| **NodeEventController** | OnPacket → ActorMessage 변환 → Queue Push |
| **QueuedResponseWriter** | Sequential 처리 보장 (Channel 기반) |
| **ActorMessageFactory** | NodePacket → ActorMessage 변환 |
| **INodeDispatcher** | MsgId → Handler 라우팅 |
| **NodeController** | [NodePacketHandler] 메서드 (비즈니스 로직) |
| **INodeResponser** | 응답 NodePacket 생성 및 전송 |

## 🎯 설계 특징

### 1. **Network Dualism 강제**
- `[NodePacketHandler]` Attribute로 Service Mesh 전용 핸들러 명시
- User Packet (`[UserPacketHandler]`)과 완전 분리

### 2. **Sequential 처리 보장**
- `QueuedResponseWriter<ActorMessage>` (Channel 기반)
- Actor Model 패턴으로 동시성 문제 방지

### 3. **Zero-Copy 최적화**
- NetMQ Msg 소유권 이전 (`Move()`)
- Span<byte> Payload 활용

### 4. **타입 안전 RPC**
- `RequestAsync<TRequest, TResponse>()` 제네릭 메서드
- 컴파일 타임 타입 검증

### 5. **자동 Handler 등록**
- Reflection 기반 Assembly 스캔
- `AddNodeControllers()` 확장 메서드로 간편 등록
- Source Generator 통합 가능 (추후 확장)

## 🔍 디버깅 포인트

### 메시지가 처리되지 않는 경우 체크리스트

1. **Controller가 등록되었는가?**
   ```bash
   # 로그 확인
   [NodeControllerExtensions] NodeController registration completed: X controllers scanned
   ```

2. **Attribute가 올바른가?**
   ```csharp
   [NodeController]  // ✅ 클래스에
   public class MyController
   {
       [NodePacketHandler(MsgId)]  // ✅ 메서드에
       public Task<IMessage> Handle(MyRequest req) { ... }
   }
   ```

3. **MsgId가 일치하는가?**
   ```csharp
   // Proto 파일과 Attribute의 MsgId가 동일해야 함
   message MyRequest { option (msg_id) = 1234; }

   [NodePacketHandler(1234)]  // ✅ 일치
   ```

4. **INodeDispatcher가 등록되었는가?**
   ```csharp
   services.AddNodeControllers();  // AddNodeControllers 호출 필수
   ```

5. **NetMQ 연결이 정상인가?**
   ```bash
   # Redis에서 Node 등록 확인
   redis-cli> HGETALL cluster:nodes
   ```

## 🚀 성능 최적화

### 1. Hot Path 최적화
- ActorMessage에 RawPayload 포함 (재파싱 방지)
- NodePacket 재구성 시 `NodePacket.Create()` 활용
- IDisposable 패턴으로 Msg 자동 정리

### 2. Memory Allocation 최소화
- `Span<byte>` 활용
- `ReadOnlyMemory<byte>` 저장
- Primary Constructor 사용

### 3. Async 최적화
- `ValueTask` 활용 가능 영역
- `ConfigureAwait(false)` 명시
- Channel<T> 기반 Queue
